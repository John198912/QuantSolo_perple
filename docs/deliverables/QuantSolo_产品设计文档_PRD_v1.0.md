## §1 产品愿景与定位

**QuantSolo** 是面向一人公司量化自营交易场景的个人 A 股全自动化交易系统。产品愿景是：

> 用纪律化、可审计的工程基础设施，在业余时间下实现专业级的量化研究与系统性交易执行，构建个人可复用的量化能力资产，在可控风险下探索正期望策略。

### 1.1 竞争定位

| 竞争对象 | 目标 | 取胜逻辑 |
|---------|------|---------|
| 广大散户（主战场） | 稳定跑赢 | 纪律化风控 + 系统化选股 + 点时正确性 |
| 小型量化公司（辅战场） | 局部差异化竞争 | LLM 舆情另类数据（待 IC 验证）+ 小资金中小盘容量优势 |
| 大型机构 | 不正面竞争 | 放弃高频/大算力，下沉中小盘，利用 20 万小资金的灵活性 |

### 1.2 产品定性

本产品**不是**一个营销驱动的金融科技产品，**不追求**短期收益最大化，**不承诺**正收益。它是一套可持续运行的个人量化基础设施，其核心产出是：

1. 高质量的点时数据管道与因子库；
2. 可复现、防过拟合的研究体系；
3. 合规、可审计的自动化执行与风控框架；
4. 基于真实行为数据的策略迭代能力。

## §2 用户画像

### 2.1 基本信息

| 维度 | 描述 |
|------|------|
| **用户类型** | 单一用户（本人）；一人公司（QuantSolo） |
| **资金规模** | 20 万 A 股（实盘最终规模），分阶段启动 |
| **投入精力** | 业余兼职，10–20h/周，中位数 **15h/周** |
| **技能背景** | Python 熟练，具备数据工程能力 |
| **组织形式** | 独立个人，单人开发、运维、交易一肩挑 |
| **交易经验** | 有 A 股投资基础，正在系统化量化转型 |
| **合规身份** | 程序化交易者（依 2025-07-07 施行细则，开户勾选并报备） |

### 2.2 用户核心诉求

| 诉求优先级 | 诉求描述 | 成功场景 |
|-----------|---------|---------|
| P0（必须满足） | 本金安全：最坏情况下最大亏损不超过 25% | 硬止损机制全程自动执行，任何情况不可绕过 |
| P0（必须满足） | 合规无忧：程序化交易报备与内部限额符合监管要求 | 开户完成勾选，首次实盘前完成报备，变更及时更新 |
| P1（重要） | 系统可信：研究结果可复现，无数据窥探偏差 | 点时正确性通过 13 场景 pytest，试验登记表完整 |
| P1（重要） | 低运维负担：每周运维成本 ≤4h，不影响主业 | 每日盘后 5–10 分钟巡检 + 周末 2h 对账 |
| P2（期望） | 正期望收益：长期实现年化 10–15% 超额收益 | 通过 B1+B3 统计闸门后，逐步加仓至 20 万 |
| P3（加分） | 差异化 Alpha：LLM 舆情因子带来可验证增量 | 舆情 IC>0.03，分层单调，扣成本为正 |

### 2.3 用户痛点与现有方案缺陷

| 痛点 | 现有方案缺陷 | 本产品方案 |
|------|------------|---------|
| 手动交易纪律差 | 情绪化操作，止损不执行 | 全自动化 + 风控守卫强制执行 |
| 个人研究过拟合严重 | 无系统防控，回测靓丽实盘亏损 | DSR + BH-FDR + 点时契约 + N_eff 预算 |
| 数据质量不可控 | 使用未来函数导致研究失效 | visible_at 重放 + 三源两票制 |
| 运维负担重 | 手动监控易遗漏，灾备无预案 | 告警自动推手机 + break-glass SOP |
| 合规意识淡薄 | 不了解程序化交易报备义务 | 合规清单内嵌产品流程，M3 前必须完成 |

## §3 经济现实（诚实版）

> **本节内容写入 PRD 正文，不得删减或美化，不得用乐观叙事覆盖。**（依据 baseline_spec.md §1 及 QS-C01 §2.2）

### 3.1 预期收益与成本

- **期望年收益（现实档）**：20 万本金 × 年化夏普 0.8 → 约 **2–3 万元/年**（粗估，非保证）；
- **运营成本**：Tushare Pro ≈ 200 元/年（2000 积分档），算力成本忽略（家用 PC），时间成本按 15h/周估算；
- **前两年 ROI 判断**：扣除数据成本、时间成本、学习曲线损耗后，**前两年 ROI 大概率为负**。这是量化系统建设的正常阶段，不代表策略失效。

### 3.2 第一目标与第二目标

| 目标等级 | 目标 | 评价标准 |
|---------|------|---------|
| **第一目标（最重要）** | 构建可复用的系统资产与研究能力 | 数据管道稳定、因子库可复现、状态机完整、对账零差错 |
| **第二目标（重要）** | 在风险约束内实现正期望收益 | 通过统计闸门后，实盘年化 >10%，夏普 >0.8 |
| **不可妥协底线** | 本金保全（25% 硬止损） | 任何情况下不允许账户回撤超过 25% |

### 3.3 诚实提示

> 不要用「验证成功后满仓」的叙事掩盖路径。**系统资产本身**——点时数据管道、因子库、状态机、对账体系——就是这个项目最核心的可交付物，无论最终实盘 Alpha 如何。（引自 QS-C01 §2.2）

### 3.4 资金分级路径（明码标价）

| 阶段 | 时点 | 资金规模 | 前提条件 |
|------|------|---------|---------|
| 模拟盘 | M3 末 | 0（模拟）| 全链路工程验收通过 |
| 小实盘 | M4 起 | 5 万（A2 降级则 2.5 万）| 合规报备完成 + 模拟盘 4 周零差错 |
| 行为观测期 | M4–M8 | 5 万不变 | B2 维持判线滚动监控 |
| 加仓窗口 | 最早 M9 | 5 万→20 万 | B1+B3 同时满足 + ≥13 周真实实盘 |

> **26 周观测窗是统计功效决定的物理下限**（QS-CAL-001 §三）：对完好策略保留率 84%，对失效策略拦截率 70%，是单位等待时间判别力最优的观测窗长度，不可用工期压缩。

## §5 产品范围——六层系统用户故事与验收标准

> 六层架构详细设计见 QS-C01 §3。本节聚焦用户价值视角，给出每层的用户故事（User Story）与验收标准（Given-When-Then 格式）。

### 5.1 数据层

**用户故事：**

> 作为策略研究者，我需要可靠、完整、点时正确的历史行情与因子数据，以便在不引入未来函数的前提下复现历史策略表现，避免研究成果上线后失效。

**功能范围：**
- AKShare / Tushare Pro / BaoStock 三源每日盘后自动采集
- 三源两票制裁决数据冲突
- visible_at 打戳与 9 表点时契约落库
- 数据质检（覆盖率、缺失率、复权因子完整性）

**验收标准（Given-When-Then）：**

| 场景编号 | Given（前提） | When（动作） | Then（预期结果） |
|---------|------------|-----------|--------------|
| D-01 | 系统初始化，无历史数据 | 首次全量采集运行 | 行情覆盖率 ≥99%，退市/ST/停复牌名录完整，visible_at 全部打戳 |
| D-02 | 每个交易日收盘后 | 盘后数据管道 17:00 触发 | 当日数据盘后 2h 内（≤19:00）完成入库，核心字段缺失率 < 0.5% |
| D-03 | 三源同一标的当日收盘价存在差异（≥0.1%） | 数据管道质检脚本运行 | 系统记录冲突详情，采用两票制取主流值，异常推送告警 |
| D-04 | 研究回测需要某标的 2019-03-15 的财报数据 | 按 visible_at 查询点时数据 | 只返回该标的在 2019-03-15 可见（公告日+1交易日）的财报快照，不返回后续修订值 |
| D-05 | 13 场景 pytest 全套运行 | CI 流水线触发 | 全部 13 场景绿色通过，无跳过，结果写入 research_ledger |
| D-06 | 数据管道运行中途网络中断 | 采集任务中断 | 系统记录断点，次日补采，Server 酱推送告警到手机，不产生半截脏数据 |

### 5.2 研究层

**用户故事：**

> 作为量化研究者，我需要一套防过拟合、可复现的因子研究框架，以便用有限的时间预算（10–20h/周）找到真实有效的 Alpha 因子，而不是把随机噪声当信号。

**功能范围：**
- 因子计算：宽进（15–20 候选）→ 严出（6–8 上线）
- 三阶段筛选：BH-FDR + t>3 → 经济显著 → Bonferroni 确认
- DSR（Deflated Sharpe Ratio）反数据窥探
- 试验登记（factor_registry + research_ledger）
- 向量化回测 + 事件驱动双层互验
- walk-forward（purged/embargo）
- LightGBM 增量训练与回退路径

**验收标准（Given-When-Then）：**

| 场景编号 | Given（前提） | When（动作） | Then（预期结果） |
|---------|------------|-----------|--------------|
| R-01 | 候选因子池 15–20 个，全部已在 factor_registry 预注册 | 三阶段筛选运行 | 对全族做 BH-FDR（q=0.10）+ \|t\|>3，不在幸存者子集上做 |
| R-02 | 线性基线回测在 validation 段运行 | 使用 cost_model_id=cm_v3_baseline | 报告夏普 ≥0.6；结果绑定 git commit hash 写入 research_ledger |
| R-03 | LightGBM 增量训练完成 | 与线性基线对比 | 夏普增量 >0.1、Calmar 增量 >0.1、特征集中度 max<40%、训练/验证 gap <15%；否则回退线性 |
| R-04 | test 段 A1/A2 终评准备就绪 | 消耗 N_eff 1 次（research_ledger 记录） | A1：年化夏普>0，A2：DSR≥0.5（N_eff≤6）；每次使用记入登记表，总次数不超过 6 次 |
| R-05 | cm_v3_advanced 成本模型就绪（M2 末） | 强制重算线性基线与 DSR | 重算后现实档夏普 ≥0.8 才可上实盘；跌破则回退研究，不得用旧结论上线 |
| R-06 | LLM 舆情因子计算完毕 | 过四条红线验证 | IC>0.03、分层单调、正交化后仍显著、扣成本为正；全部通过才纳入模型，任一不过则推迟至 M4+ |

### 5.3 信号层

**用户故事：**

> 作为交易执行者，我需要每个交易日收盘后得到明确的目标持仓表（持什么、多少），以便执行层无歧义地执行调仓操作，不因信号模糊产生执行偏差。

**功能范围：**
- 核心多因子选股（Top 10–15 只，波动率倒数加权，75–80% 仓位）
- 趋势卫星（唐奇安 20 日突破 + 2×ATR 跟踪止损，20–25% 仓位）
- 大盘择时总开关（沪深 300 vs 200 日 MA，三档 90–100%/60%/30%）
- 目标持仓表生成与信号合并

**验收标准（Given-When-Then）：**

| 场景编号 | Given（前提） | When（动作） | Then（预期结果） |
|---------|------------|-----------|--------------|
| S-01 | 盘后数据管道完成，因子计算完毕 | 信号层运行（核心多因子） | 输出 Top 10–15 只目标持仓，含波动率倒数权重，剔除 ST/停牌/上市<250日/日均成交额<5000万 |
| S-02 | 大盘择时：沪深300连续 N 日跌破 200 日 MA | 择时总开关评估 | 总仓位上限切换至 30%，信号按比例缩减；N 日确认延迟防假信号 |
| S-03 | 趋势卫星持仓触发 2×ATR 止损 | 卫星止损信号生成 | 当日生成卖出信号，写入目标持仓表，执行层在下一交易日开盘处理 |
| S-04 | 核心多因子组合换股（原持仓 A 换为 B） | 目标持仓表更新 | 新目标持仓表包含完整的持有/买入/卖出三类操作，权重之和 = 当前仓位上限 |
| S-05 | 模拟盘阶段，盘前 9:15 dry-run | 信号层重跑 | 输出与前日信号的 diff，标注新增/退出/维持，供人工核查 |

### 5.4 风控层

**用户故事：**

> 作为账户持有人，我需要风控层无条件凌驾于所有策略之上，确保任何情况下都不会出现超过硬止损线的损失，哪怕我在睡觉时系统自动运行。

**功能范围：**
- 下单唯一守卫（铁律一，见 QS-C01 §1）
- 三级回撤：20% 预警→降仓 50%；25% 硬止损→全清仓冻结
- 日常硬约束：单票 ≤8%（实盘 5 万阶段放宽至 ≤20%）、单行业 ≤30%、流动性过滤
- 内部限额：1 笔/秒、200 笔/日（远低于监管高频认定线）
- 物理熔断（break-glass，独立进程，直接调 xtquant）
- 状态机 15 态管理（全文见 QS-C04）

**验收标准（Given-When-Then）：**

| 场景编号 | Given（前提） | When（动作） | Then（预期结果） |
|---------|------------|-----------|--------------|
| FC-01 | 任何策略信号触发下单 | 下单请求发出 | 必须经过 risk_guard 前置校验；校验未通过则记录 log_risk_block，不发出实际订单 |
| FC-02 | 账户回撤达到 20% | 任意交易时点 | 系统强制降仓至总净值 50%，优先卖出卫星仓，推送告警，写入 execution_ledger |
| FC-03 | 账户回撤达到 25% | 任意交易时点 | 系统全部清仓，冻结交易，触发人工复盘通知，写入 execution_ledger，状态机进入 HALT |
| FC-04 | 策略试图对单只标的买入超过净值 8% | 下单前校验 | 风控守卫拦截，自动截断至 8% 上限，记录拦截日志 |
| FC-05 | QMT 断连，状态未知 | 执行守护进程检测到失联 | 状态机进入 BREAK_GLASS，物理熔断脚本就绪，推送紧急告警，等待人工介入 |
| FC-06 | 研究调试模式下需要手动测试下单 | 调试人员调用非标准路径 | 系统拒绝，提示必须通过 place_order 统一入口，不允许任何路径绕过风控守卫 |
| FC-07 | 单日累计申报达到 200 笔 | 后续信号触发下单 | 内部限额熔断，拒绝本日后续下单，推送告警 |

### 5.5 执行层

**用户故事：**

> 作为系统运营者，我需要执行层在合理的时间窗口内完成调仓操作，处理好涨跌停/停牌等特殊情况，并保留完整的执行审计记录，以便事后对账和问题排查。

**功能范围：**
- 三层隔离架构：策略逻辑层 / 适配器层 / xtquant（全文见 QS-C01 §10.1）
- 国金 miniQMT 主力路径（入金 10 万保持 3 个月后永久开通）
- 拆单逻辑（小盘股分批，降低冲击成本）
- 下单重试与涨跌停/停牌处理
- 幂等键去重（防重复下单）
- execution_ledger 完整记录

**验收标准（Given-When-Then）：**

| 场景编号 | Given（前提） | When（动作） | Then（预期结果） |
|---------|------------|-----------|--------------|
| E-01 | 策略逻辑层输出目标持仓表 | 执行层调用 place_order | 订单经风控签名 + client_order_id + 幂等键 + 目标持仓快照写入 execution_ledger |
| E-02 | 下单成功后 QMT 返回成交回报 | 执行层接收回报 | 更新 execution_ledger 对应记录，状态从 SUBMITTED 变更为 FILLED/PARTIAL_FILLED |
| E-03 | 目标标的当日涨停 | 执行层尝试买入 | 系统挂涨停价排队，不强行破价；记录排队状态；次日重评是否继续 |

## §8 产品成功指标（北极星）

> **核心理念**：本产品的北极星指标**不是**短期收益率，而是**系统可信度与研究能力的可持续积累**。（依据 baseline_spec.md §1）

### 8.1 北极星指标定义

**北极星 = 模拟盘 + 实盘行为数据质量**

具体构成：
1. **对账零差错率**：execution_ledger 与 QMT 持仓对比的差异发生率（趋向 0）；
2. **B3 合规天数**：连续满足 B3 工程判线（对账零差错、成本偏差 ≤30%、风控行为一致）的累计天数；
3. **rank-IC 数据积累质量**：模拟盘/实盘 rank-IC 序列的完整性与记录规范度（每周完整记录，无缺失）；
4. **N_eff 预算使用效率**：6 次 test 评估预算的消耗速度与决策质量（不浪费在方向不对的试验上）。

### 8.2 分阶段关键指标

| 阶段 | 关键指标 | 目标值 | 监控频率 |
|------|---------|--------|---------|
| M1 完成 | pytest 13 场景全绿率 | 100% | 每次代码提交 |
| M1 完成 | 数据管道成功率（连续 20 交易日）| ≥95% | 每日 |
| M2 完成 | 线性基线夏普（validation + cm_v3_baseline）| ≥0.6 | 单次评估 |
| M2 完成 | test 段 A1/A2 评估通过 | A1 夏普>0 | 单次评估 |
| M3 完成 | 模拟盘对账零差错连续天数 | ≥20 交易日 | 每日 |
| M3 完成 | 告警推送延迟 | < 5 分钟 | 每次演练 |
| M4 启动 | 合规报备材料完整性 | 100% 条目完成 | M4 前一次性 |
| M4–M8 | B2 维持判线状态 | 每周滚动 26 周 IC 均值 > 研究IC−1.645×SE | 每周 |
| M9+ | B1 加仓判线状态 | 滚动 26 周 IC > 研究IC−1.0×SE，且 ≥13 周真实实盘 | 每周 |

### 8.3 警示指标（Leading Indicators of Failure）

> 以下指标下行时提前预警，无需等待账户亏损才采取行动：

| 警示指标 | 警戒值 | 响应动作 |
|---------|--------|---------|
| 滚动 26 周 rank-IC 均值 | 跌破 研究IC−1.645×SE（B2 判线） | 降仓，触发策略复盘 |
| 实际成本偏差 | > +30%（实测/建模，B3 要求）| 核查成本模型，暂停扩仓 |
| 对账差异连续天数 | ≥5 天未归零 | 暂停自动化，人工排查 |
| 运维时间 | > 4h/周 持续 2 周 | 触发砍范围评审 |
| 账户回撤 | 接近 20%（预警线）| 检查策略状态，准备手动干预预案 |

## §10 反范围（明确不做的事）

### 10.1 技术反范围

以下技术方向在本版本（M1–M3，含 M4 启动后初期）明确不做：

| 不做的事 | 原因 | 替代方案 |
|---------|------|---------|
| 强化学习（RL/DRL）| A 股非平稳，样本不足，个人 DRL 实盘投入产出比极低（QS-C01 §8.1）| LightGBM 非线性合成 |
| 深度学习（LSTM/Transformer）| 等价独立样本约 20–50 万，调参成本超出 10–20h/周 预算 | LightGBM + 线性基线 |
| 迁移学习（跨市场）| A 股 regime 切换频繁，负迁移风险高 | 仅 LLM 文本层隐式受益 |
| PostgreSQL 双库 | 一人公司零运维原则，DuckDB/SQLite 足够 | DuckDB + SQLite 两层 |
| 高频交易策略 | 监管、技术、资金规模均不适合 | 中低频（核心周度 + 卫星 2–10 天）|
| 北向资金个股择股 | 2024-08-19 已停止披露 | 龙虎榜 + 主力资金流 + 股东户数 |
| 多账户/多策略并发 | 一人公司单账户，超出运维能力 | 单账户单策略 |
| 外部云服务器部署 | QMT/xtquant 仅支持 Windows，家用 PC 已满足 | 家用 Windows 11 PC |

### 10.2 数据反范围

| 不做的事 | 原因 |
|---------|------|
| Tushare 5000 积分 THS 资金流（P2 阶段前）| M2 后评估，不进 M1/M2 预算 |
| 境外市场数据采集 | 合规复杂，超出产品定位 |
| Level-2 逐笔/盘口数据 | 成本高，中低频策略不需要 |
| 实时舆情监控（P1 阶段前）| P2 任务，M3 后评估 |

### 10.3 超支砍范围序列（按顺序执行）

> 当工时超支 20%（即 15h/周基准下，单月实际工时 > 72h，或 3 个月总工时 > 252h）时，**必须按以下序列砍范围**，不得靠增加运维时间应对：

| 步骤 | 砍什么 | 影响 | 何时恢复 |
|------|-------|------|---------|
| 第一刀 | LLM 舆情因子推迟至 M4+ | 主策略不受影响，仅差异化 Alpha 延后 | M4 有余力时重评 |
| 第二刀 | LightGBM 留线性基线（夏普 0.6 可放行）| 上线策略变为纯线性，不影响合规和风控 | M4 后有余力时补做 |
| 第三刀 | M3 模拟盘延长至第4个月 | 推迟实盘但不降低质量 | 模拟盘验收通过即启动 M4 |

## 附录A 术语表

| 术语 | 定义 |
|------|------|
| visible_at | 数据在某交易日可被策略合法使用的时间戳；点时正确性的核心字段（详见 QS-C03）|
| N_eff | 等价独立试验次数，用于 DSR 计算；本项目全生命周期封顶 6 次 test 评估（QS-CAL-001）|
| rank-IC | 截面收益排名与因子值排名的相关系数；B 类闸门的核心统计量 |
| SE | 标准误（Standard Error），IC 序列的均值标准误，用于 B1/B2 判线 |
| DSR | Deflated Sharpe Ratio，纳入试验次数惩罚的夏普比率（Bailey–López de Prado 公式）|
| trigger_ic_audit | 背离触发词，唯一定义见 QS-C02 §7.x（bootstrap CI 口径），本文不重抄 |
| BREAK_GLASS | 状态机紧急态，跳过策略层直接平仓；15 态之一（QS-C04）|
| cost_model_id | 回测成本模型标识；cm_v3_baseline（第2月）vs cm_v3_advanced（第3月起及上线）|
| B1 判线 | 加仓主裁决：模拟盘+5万阶段 ≥26 周，实测 rank-IC > 研究IC−1.0×SE（QS-CAL-001）|
| B2 判线 | 维持判线，终身滚动：滚动26周 IC > 研究IC−1.645×SE 且 >0（QS-CAL-001）|
| B3 判线 | 工程判线：成本偏差≤30%、对账连续4周零差错、风控行为一致（QS-CAL-001）|
| miniQMT | 迅投 QMT 轻量版，国金证券入金 10 万保持 3 个月后永久开通 |
| MoSCoW | 需求优先级框架：Must（必须）/ Should（应该）/ Could（可以）/ Won't（不做）|
