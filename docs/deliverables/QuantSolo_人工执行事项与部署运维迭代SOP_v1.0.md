## 三、P0：开工前准备（目标 ≤1 周完成）

### 3.1 硬件与物理环境 🔑

| 事项 | 规格/标准 | 验证点 |
|---|---|---|
| 主机 | Windows 11 专业版；≥6 核 / 32GB / 1TB NVMe（可用现有 PC，先跑 M1，回测峰值不足再升级） | 系统更新设为「暂停盘中重启」 |
| UPS | ≥600VA 在线互动式，仅接主机+路由器 | 拔电测试：支撑 ≥10 分钟，足够撤单清仓 |
| 网络 | 有线宽带为主；手机热点写入灾备路径 | 双链路均可登录券商 |
| 外置备份盘 | ≥1TB，仅备份用 | 首次手动备份成功 |
| 物理位置 | 主机常开不休眠；电源/网线物理固定 | 电源计划=高性能，禁用睡眠 |

### 3.2 账户与服务开通 🔑

| 事项 | 操作要点 | 时限 |
|---|---|---|
| **国金证券开户** | 线上开户→风险测评选**激进型**→开户问卷**如实勾选程序化交易**→入金 10 万→申请 miniQMT 权限 | **第 1 天启动**：10 万保持 3 个月才转永久权限，这是全项目最长外部计时器（E06 §3.3 五步表） |
| Tushare Pro | 注册+购买 2000 积分档（约 200 元/年） | M1-W1 前 |
| AKShare / BaoStock | 免费无需账号，pip 安装即用 | 随仓库初始化 |
| Server酱 或 钉钉机器人 | 注册并取得 SendKey/Webhook，手机端确认能收到测试消息 | M1 内完成即可，M3 前必须 |
| 云盘 + rclone | 任选（阿里云盘/OneDrive 等），配置 rclone 授权 | M1 内 |
| GitHub 私有仓库 | 创建 repo，把 QS 全部文档放入 `docs/`，QS-E07 复制为根目录 `AGENTS.md` | 开工前 |

### 3.3 密钥与安全基线 🔑

1. 新建本机独立 Windows 账户跑交易进程；券商密码、Tushare token、Webhook key 全部进 Windows 凭据管理器或 `.env`（加入 `.gitignore`），**绝不进 Git**。
2. 备份加密口令离线记录（纸面+一处异地）。
3. 向 Agent 提供的环境中**永不包含**券商交易密码——实盘登录始终由你在 QMT 终端人工完成。

### 3.4 P0 完成判据

### 7.2 告警响应约定 🔁

- **P1（立即响应，≤15 分钟）**：25% 硬止损触发、下单失败连续、进程失联、UPS 切换 → 按 E05 §5 对应 SOP；联系不上系统状态时直接走熔断 SOP。
- **P2（当日处理）**：20% 预警降仓确认、数据管道失败（盘后有整夜缓冲）。
- **不可值守日**（出差/休假）：提前一晚把系统切到「只平不开」或全平模式（C04 暂停态），**宁可错过行情，不可无人值守持仓**——这是铁律五的具体化。

## 八、产品迭代期 SOP（贯穿全程）

### 8.1 因子/策略迭代（常规迭代，不动宪法）

1. 新因子想法 → 记入研究 backlog（不立即开发）；每月集中评审一次，控制研究 WIP ≤2。
2. 开发与筛选全程只用 train+validation（validation 查看计入 ≤5 轮预算，C02）。
3. 入库标准：三阶段筛选+BH-FDR 通过 → 进因子库 observation 状态，**不直接进实盘组合**。
4. 实盘组合调整 = 版本发布事件（§8.3），每季度至多一次，杜绝周度手痒调仓。
5. 因子退役：连续 N=3 个评估周期失效 → 退役归档（C01 §15），退役记录不可删。

### 8.2 宪法修订（动闸门/铁律/冻结参数）

严格走 C00 §四六步流程：触发→提案（现条文/问题/新条文/影响面）→**证据（动闸门数字必须重跑 `gate_calibration.py` 附功效表）**→48h 冷静期→全文版落地+索引版本锁定矩阵更新→Git tag。频率 ≤1 次/月；回撤期/连败期**禁止**修宪（情绪化修宪是单人项目第一大坑）。

### 8.3 版本发布流程

1. 发布窗口：仅周末（非交易时段）；发布前全量回归（13 场景+单元+集成）绿。
2. 发布三件套：代码 tag + 文档同步版本 + CHANGELOG 条目（Agent 维护，你审核）。
3. 涉及下单逻辑/风控/申报行为的变更 → 先模拟盘跑 ≥1 周再切实盘，并评估是否触发报备更新（E05 §9 判定表）。
4. 回滚预案：每次发布前确认上一 tag 可一键回滚，数据库 schema 变更必须向后兼容一个版本。

### 8.4 年度审查（每年 1 月）

☐ 全文档体系通读纠漂移 ☐ 合规细则年检（监管是否有新规） ☐ 数据源健康年检 ☐ 经济现实复盘（ROI、是否继续/扩大/收缩） ☐ N_eff 预算余额盘点 ☐ 硬件寿命检查（盘/UPS 电池）。
