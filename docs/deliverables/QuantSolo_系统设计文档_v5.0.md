## §1 五条铁律

> **本节是五条铁律的全文唯一所在地（QS-C01 §1）。其他所有宪法文档与工程文档只引用编号，不重抄内容。**

### 铁律一：风控凌驾策略，下单唯一入口必经守卫

任何策略信号都必须先通过风控层校验才能转化为订单。风控层实现为下单函数的强制前置守卫（装饰器/拦截器），任何路径——包括紧急操作、研究调试、手工指令——都不得绕过此守卫直接下单。物理一键熔断（独立进程，直接调 xtquant 撤单+市价平仓）是唯一允许绕过策略逻辑的例外，但仍须写入执行审计日志。

### 铁律二：研究纪律——冻结样本、试验登记、N_eff 预算

- **样本冻结**：train 2016–2021 / validation 2022–2023 / test 2024–2025 三段物理隔离，test 段在整个研发期一次都不能看，只在上实盘前跑一次终评。
- **试验登记**：所有对 test 段的评估均须事前记入试验登记表（`factor_registry` + `research_ledger`），未登记的评估结果不具备放行效力。
- **N_eff 预算**：整个项目生命周期 test 段评估次数封顶 6 次（QS-CAL-001 定标）；validation 阶段查看预算 ≤5 轮（旧规保留）；用满后只能动用 2026 年起新样本。

### 铁律三：点时正确性——一切研究数据按 visible_at 重放

研究与回测中任何数据的使用，只能以该数据在当时交易日可见（visible_at）的版本为准，严禁使用事后修订数据。点时契约 canonical 排序四键不变：`visible_at DESC, revision_seq DESC, snapshot_rank DESC, record_id DESC`；修改=追加ACTIVE、撤销=追加VOIDED。所有模块接口必须携带 `visible_at, ingested_at, source, snapshot_id, calendar_version, corp_action_known_at` 六个点时元数据字段。

### 铁律四：资金分级——5万（或2.5万）→行为闸门→20万

实盘资金分三级启动：
1. **模拟盘阶段**（M3 末）：全链路上线，行为数据开始累积；
2. **小实盘阶段**（M4 起）：5万（若 A2 触发降级则 2.5 万）真实实盘，触发 B 类行为闸门观测；
3. **加仓阶段**（最早 M9）：B1+B3 同时满足，5万→20万，观测窗 26 周物理下限不可压缩；加仓前至少 13 周必须来自真实 5 万实盘。

### 铁律五：每日人工巡检 + 周度对账，自动化 ≠ 无人值守

自动化系统不免除人工监督义务：
- **每日**：盘后 5–10 分钟人工巡检（核对持仓与目标、检查告警、确认数据更新成功）；
- **每周**：2 小时对账复盘（因子 IC 走势、成本偏差、风控触发记录、B2 维持判线状态）；
- **总运维预算 ≤4h/周**，超出即触发简化评审（削减范围，非增加人力）。

## §2 项目定位与经济现实

### 2.1 目标用户画像

| 维度 | 内容 |
|------|------|
| 本金 | 20 万 A 股 |
| 投入精力 | 业余兼职，10–20h/周，中位数 15h/周 |
| 技能 | Python 熟练，具备数据工程能力 |
| 组织形式 | 一人公司（QuantSolo），单人开发运维 |

### 2.2 经济现实（诚实版，不回避）

**本段内容写入 PRD，不得删减或美化：**

- 20 万本金 × 年化夏普 0.8 → 期望年收益约 **2–3 万元**（现实档估算）；
- 扣除数据成本（Tushare Pro ≈200元/年）、算力与时间成本后，**前两年 ROI 大概率为负**；
- 项目**第一目标**：构建可复用的系统资产与研究能力；盈利是第二目标；
- **本金保全（25% 硬止损）是不可妥协底线**，非收益优化项。

> **诚实提示：** 不要用"验证成功后满仓"的叙事掩盖路径。系统资产本身——点时数据管道、因子库、状态机、对账体系——就是这个项目最核心的可交付物，无论最终实盘 Alpha 如何。

### 2.3 竞争定位

| 竞争对象 | 目标 | 取胜逻辑 |
|---------|------|---------|
| 广大散户（主战场） | 稳定跑赢 | 纪律化风控 + 系统化选股 + 点时正确性 |
| 小型量化公司（辅战场） | 局部竞争 | LLM 舆情另类数据（待 IC 验证）+ 小资金中小盘容量优势 |
| 大型机构 | 不正面竞争 | 放弃高频/大算力，下沉中小盘，利用资金量小的灵活性 |

## §3 系统架构总览

### 3.1 六层分层架构

| 层 | 职责 | 关键组件 | 运行时点 |
|----|------|---------|---------|
| **数据层** | 采集、清洗、入库，防未来函数 | 龙虎榜 / 资金流 / 行情 / 财报 / 舆情 → DuckDB + Parquet + SQLite | 每日盘后 17:00 |
| **研究层** | 因子挖掘、回测、防过拟合 | 因子库 + LightGBM + walk-forward 回测 | 离线（非交易时段）|
| **信号层** | 选股、择时、信号合并 | 核心多因子 + 卫星突破 + 大盘择时 → 目标持仓表 | 盘后/盘前 |
| **风控层** | 强制校验，凌驾策略 | 仓位/行业上限 + 三级回撤 + 流动性过滤 | 每笔下单前（硬中断）|
| **执行层** | 下单、拆单、重试 | 策略逻辑层 / 适配器层 / xtquant | 盘中（自动）|
| **监控层** | 盈亏、告警、留痕 | Streamlit 看板 + Server酱/钉钉机器人 + 全量日志 | 实时 |

### 3.2 核心架构约束

```
策略逻辑（纯 Python，平台无关）
        │
        │ 统一数据/订单接口（抽象层）
        ├──────────────────────────────┐
        ▼                              ▼
  回测适配器                       实盘适配器
  → 历史 Parquet                  → QMT (xtquant)
                                         │
                                  风控守卫（唯一下单入口）
                                         │
                                    execution_ledger
```

**三层隔离原则**：策略逻辑层禁止 `import xtquant`；所有平台相关调用封装在适配器层；xtquant 调用只在执行层出现。回测验证过的逻辑可无缝上实盘，无需重写。

## §4 数据层

### 4.1 北向数据修正（致命修正）

> **事实：** 自 **2024-08-19** 起，沪深交易所调整沪深港通信息披露机制，北向资金个股流向数据停止公开披露。AKShare `stock_hsgt_hist_em` 接口此后静默返回空值，不报错。**原依赖北向个股择股的任何逻辑均已从数据源断裂，必须完全废除。**

**「聪明钱」主线迁移：**

| 废除 | 替代 |
|------|------|
| 北向资金个股流向（已停披） | **龙虎榜**（游资动向、情绪过滤）|
| — | **个股主力资金流**（主力大单净流入，AKShare 东财源）|
| — | **股东户数环比**（筹码集中度，Tushare Pro）|

北向数据仅保留「沪深股通前十大活跃成交股」与季度持仓变动，降级为 P3 辅助参考，不作选股主线。

### 4.2 三源选型（最终决策）

| 数据源 | 角色 | 成本 | 备注 |
|--------|------|------|------|
| **AKShare** | **主采集** | 免费 | 日线行情（后复权）、龙虎榜、个股资金流（东财）、停复牌/ST/退市名录 |
| **Tushare Pro** | **对账与补强** | 2000积分档约200元/年；5000积分档另需年费 | 日线/财报/龙虎榜明细/股东户数作为第二来源交叉校验；5000积分(THS资金流)暂不买，P2 再评估 |
| **BaoStock** | **冗余校验** | 免费 | 日线第三源，三源两票制裁决数据冲突 |

**积分成本事实（原方案「¥200/年」不准确）：**
- 龙虎榜明细 / 机构明细 → 需 **2000 积分**
- 个股资金流（moneyflow）→ 需 **2000 积分**
- 行业及个股资金流（THS：moneyflow_ths/ind_ths）→ 需 **5000 积分**（通常需年费，约数百元）

**舆情源（P2，第 3 月后）：** 公开新闻 + 研报，LLM（本地 Qwen 或低价 API）结构化打分，Parquet 缓存。

### 4.4 点时契约存储设计

**存储分层（砍掉 PostgreSQL）：**

| 存储形态 | 内容 | 说明 |
|---------|------|------|
| **Parquet + DuckDB** | 行情面板、因子面板、舆情缓存 | 列存、查询快、零运维、单机充足；`adj_factor_pit` 大表按年分区 Parquet |
| **SQLite** | 交易/持仓/信号/审计（9表+trade_calendar）| 事务安全、便于审计 |

> **明确反对 PostgreSQL 双库方案：** 一人公司零运维原则。20 万中低频系统的瓶颈是 Alpha 和纪律，不是数据库吞吐。点时契约 9 表 + trade_calendar 全部落 DuckDB/Parquet + SQLite 两层。

**点时规则（visible_at 统一口径）：**
- 盘后采集统一打当日 17:00 采集时间戳为 `visible_at`；
- 财报用公告日期 + 1 交易日（保守处理）；
- 新闻用发布时间戳：t−1 日 15:00 至 t 日 15:00 归 t+1 交易日（防未来函数）。

**点时契约 9 表：** `security_master`、`daily_bar_raw`、`adj_factor_snapshot`、`corporate_action`、`st_status_history`、`suspension`、`limit_price`、`financials_pit`、`factor_snapshot`；加 `trade_calendar`（交易所官方日历，处理调休补班）。

**模块接口点时元数据（必带六字段）：**
`visible_at`、`ingested_at`、`source`、`snapshot_id`、`calendar_version`、`corp_action_known_at`

**因子变体三枚举（与 QS-C03 对齐）：**
- `processed`：去极值 + 行业市值中性化 + z-score，供 Ridge 线性训练；
- `orthogonal`：Cholesky 残差化，供 LightGBM 训练；
- `raw`：原始未变换，供 SHAP 归因与人工解读（标注"原始因子重要性近似"）。

## §5 策略层

### 5.1 组合结构

| 子策略 | 仓位占比 | 调仓频率 | 核心逻辑 |
|--------|---------|---------|---------|
| **核心多因子** | **75–80%** | 周度 | 多因子打分 → Top10–15 只 → 波动率倒数加权 |
| **趋势卫星** | **20–25%** | 持仓 2–10 天 | 唐奇安 20 日突破 + 2×ATR 跟踪止损 |
| **大盘择时总开关** | — | N 日确认延迟 | 沪深 300 vs 200 日 MA，三档 90–100%/60%/30% |

### 5.2 核心多因子（75–80%，周度）

**因子库（研究池 15–20，生产上线 6–8）：**

| 因子类别 | 具体因子 | 方向 | 数据依赖 |
|---------|---------|------|---------|
| 质量 | ROE、毛利率、经营现金流/净利润、资产负债率 | 高质量正向 | 财报（P1）|
| 动量 | 20/60 日收益（剔除最近 5 日反转）| 适中偏正 | 行情（P0）|
| 低波 | 60 日波动率、最大回撤 | 越低越好 | 行情（P0）|
| 筹码/资金 | 主力净流入、股东户数环比、龙虎榜净买 | 正向 | 资金流（P0/P1）|
| 情绪（P2）| LLM 舆情正交化得分 | 正向 | 舆情（P2）|

**选股流程：**

```
全市场
→ 剔除 ST / 停牌 / 上市<250日 / 日均成交额<5000万
→ 各因子 MAD 去极值 + 市值/行业中性化 + z-score
→ 两条打分路径并行：
    A. 线性加权打分（基线，可解释，Ridge）
    B. LightGBM 排序模型（非线性增强）
→ 融合排序（取平均排名）→ 选 Top 10–15 只
→ 波动率倒数加权（低波多配）
```

**20 万资金的小盘优势与陷阱：** 可适当下沉到中小盘，博取大资金做不了的容量空间。但**必须坚持日均成交额 ≥5000 万硬门槛**，否则卖出会自己砸盘。回测须按真实成交额对冲击成本建模，不能假设瞬时全额成交。

**因子共线性处理：**
- 只对高相关簇做残差化（非全量 Cholesky）；
- Spearman 相关矩阵 → r>0.7 保留 IC 高者 → VIF>5 人工审查 → 高相关簇残差化 → 记录变换矩阵；
- 训练用双集（processed + orthogonal）；归因用 raw 一份。

### 5.3 趋势卫星（20–25%，持仓 2–10 天）

- **入场：** 价格突破 N 日唐奇安通道上轨（N=20）+ 量能放大确认（可选回踩确认）；
- **出场：** 跌破 2×ATR 跟踪止损，或突破失败；
- **标的：** 个股 + 行业 ETF（ETF 无涨跌停撤单风险，更适合突破执行；ETF 仍有涨跌幅/流动性/折溢价/跟踪误差，非无风险）；
- **T+1 处理：** 持仓 ≥2 天，天然规避 T+1 限制。

### 5.4 大盘择时总开关

| 市场状态 | 判断条件 | 总仓位上限 |
|---------|---------|---------|
| 牛市 | 沪深 300 在 200 日 MA 上方 + 量能健康 | 90–100% |
| 震荡 | 价格在 MA 附近反复 | 60% |
| 熊市 | 跌破 200 日 MA + 量能萎缩 | 30% |

**择时定位注意：** 择时仅用作极端风险的总开关，不做频繁加减仓。信号需连续 N 日成立才切换档位（N 待研究期 A/B 验证确定），降低假信号导致来回打脸。如关闭择时后夏普更高，则削减择时权重（强制 A/B 验证）。

## §6 因子研究框架

### 6.1 三本账基础设施

| 账本 | 记录内容 | 作用 |
|------|---------|------|
| `factor_registry` | 每个因子的假设、方向、变体数、测试状态 | 试验次数 N 的来源，喂给 Deflated Sharpe |
| `research_ledger` | 每次回测的数据快照、git commit hash、参数、结果 | 可复现、代码版本与回测结果绑定 |
| `execution_ledger` | 目标持仓、风控裁剪、订单、成交、对账差异 | 实盘审计与对账 |

**试验登记铁律：** 显式记录测试过的因子/参数/模型总数，该 N 直接进入多重检验惩罚公式。这是个人量化最容易偷懒、也最致命的环节。

### 6.2 三阶段因子筛选（修复多重检验漏洞）

**关键修复：多重检验必须对全因子族做，不在幸存者子集上做。**

**阶段一：统计显著**
1. 对全部预注册因子 M_registered 计算 p 值；
2. BH 过程找最大 j 使 pⱼ ≤ (j/M)×q，q=0.10；
3. 叠加 |t|>3 条件（HAC/Newey-West 修正后）；
4. **通过** = BH 显著 AND |t|>3。

> t 统计量定义：因子方向预注册，负向因子先方向归一化，IC 序列用 HAC/Newey-West 修正或 block bootstrap 处理自相关（普通 OLS t 会高估显著性）。

**阶段二：经济学显著**

| 条件 | 阈值 |
|------|------|
| 单因子 IC 均值 | > 0.02 |
| IC 符号稳定率 | > 60% |
| 分组多空收益 | Top − Bottom > 0 |

**阶段三：最终上线确认（Bonferroni）**
- p < 0.05 / M_registered（用原始测试数，非上线数 K）；
- 经济上保留 6–8 个进入生产。

> 闭环：研究池宽进（15–20）→ 阶段一（全族 BH-FDR + t>3）→ 阶段二（经济显著）→ 阶段三（M_registered 级 Bonferroni）→ 严出（6–8 上线）。

### 6.3 Deflated Sharpe Ratio（DSR）

把「一共试了多少因子/参数/模型组合」的试验次数 N 纳入惩罚——单纯提高 t 阈值无法消除「同一数据反复试验」的数据窥探偏差。DSR 按 Bailey–López de Prado 公式，SR0 取 N_eff 次有效试验的期望最大夏普（Harvey–Liu 口径），偏度/峰度用样本矩代入。

### 6.4 成本档一致性（v4.1 修订）

- `cost_model_id = cm_v3_baseline`：常态滑点，第 2 月线性基线放行用；
- `cost_model_id = cm_v3_advanced`：进阶动态滑点，第 3 月起及上线用；
- **强制重判：** `cm_v3_advanced` 就绪后，线性基线与 DSR 必须用 `cm_v3_advanced` 重算一次；重算后跌破现实档 0.8 → 回退研究，不得用旧结论上线；
- 上线、模拟盘对账、最终研究判定三处的 `cost_model_id` 必须全部为 `cm_v3_advanced`。

### 6.5 背离定义（唯一口径，v4.1 修订）

**显著背离（trigger_ic_audit）唯一定义：**  
`realized_ic < research IC 的 block bootstrap 5% 分位下界`（滚动 26 周实测 IC 均值落在研究期 IC bootstrap 90% CI 下界之外）且 `n_fills ≥ 50`。

废除「连续 8 周 < 中位数 30%」旧定义。

**5 万阶段特例：** 约 5 只持仓，成交不足 50 笔时，背离统计项标注 `INSUFFICIENT_NOT_BLOCKING`，加仓判定仅用三项工程门槛（对账/成本/风控），背离项不阻塞但留痕。

### 6.6 行业市值中性化 A/B 验证

回测对比「中性化 vs 不中性化」Alpha 留存——中小盘下沉策略的核心 Alpha 可能来自市值因子，强行中性化可能把收益源中性化掉。

| 场景 | 处置 |
|------|------|

## §8 ML模型选型论证

### 8.1 不上 RL/DL 的决策根据

**核心事实：A 股的等价独立样本很小**

A 股约 5000 只股票 × 10 年 ≈ 1200 万行面板，看似海量。但**有效独立样本远小于此**：同一截面上个股横向高度相关（等价样本要除以明显大于 1 的因子），再叠加 5–20 日标签窗口（相邻样本标签相关），**等价独立样本可能只有原始的 1/20～1/50，即约 20–50 万行**。

| 模型类 | 判断 | 核心理由 | 本系统决策 |
|--------|------|---------|---------|
| 线性 / Ridge 多因子 | ✅ 必做基线 | 样本利用率高、可解释、抗过拟合；天然兼容 Ridge 对 A 股多动量因子共线性稳健 | 永不删除，第一性基线 |
| GBDT (LightGBM) | ✅ 值得 | 正好落在等价独立样本的稳定区间；捕捉因子非线性交互；SHAP 可归因 | 中等深度（叶子 64–128），叠在线性之上 |
| 深度学习（LSTM/Transformer）| ⚠ 暂不 | 独立样本不足以支撑参数量，极易过拟合；调参与维护成本高，与 10–20h/周不匹配 | 暂缓；不进实盘 |
| 强化学习（DRL）| ❌ 不做 | 金融环境非平稳，违背 RL 收敛假设；奖励稀疏且噪声极大；回测里极易「记住」历史路径 | 放弃；不为「先进感」买单 |
| LLM（舆情）| ✅ 唯一真差异化 | 小型量化难做的另类数据；IC 约 0.02–0.04，须严格证伪 | 重点投入，见 §9 |

### 8.2 RL 不划算的具体论证

1. **强化学习：** 金融市场是非平稳环境（分布随时间漂移），违背 RL 收敛的基本假设；奖励信号稀疏且噪声极大；在回测里 RL 极易「记住」历史路径而非学到泛化策略，实盘后迅速失效。学界综述与从业者社区普遍认为个人 DRL 实盘投入产出比极低。

2. **深度学习：** LSTM/Transformer 在价量序列上的边际收益，在样本受限的 A 股很难稳定超过 LightGBM，且过拟合风险随参数量陡增。对一人量化而言，同样时间投入数据质量 + 因子构造 + LLM 舆情的产出远高于 DL 调参。

3. **迁移学习：** 在跨市场/跨周期迁移上理论有价值，但个人难以获得高质量源域数据，且 A 股 regime 切换频繁，负迁移风险高。建议仅在 LLM 文本模型层面隐式受益（用预训练大模型做情绪抽取），不在价量模型上专门做迁移。

### 8.3 LightGBM 增量验收标准

| 指标 | 标准 |
|------|------|
| 夏普增量 | > 线性 + 0.1 |
| Calmar 增量 | > 线性 + 0.1 |
| 特征重要性集中度 | max < 40%（防单因子依赖）|
| 参数稳定性 | 参数 ±20% 下 Top10 重合 > 70% |
| 训练/验证 gap | < 15%（防过拟合）|

**LightGBM 回退路径：** 线性通过但 LGB 不通过 → 直接线性上线，不设 LGB 依赖。

### 8.4 一句话选型结论

> **不上 RL、暂不上 DL。** ML 预算全部押在两处：① LightGBM 做因子非线性合成；② LLM 做舆情情绪因子（真差异化）。这与一人量化的时间预算和 A 股数据现实最匹配，也最不容易自欺。

## §9 LLM舆情差异化路线

> **本节是 QuantSolo 唯一真正差异化的 Alpha 来源**。大型量化公司的优势在算力和数据，小型公司的壁垒在 LLM 对另类文本数据的结构化处理。但必须用严谨的 IC 检验证明它真有 Alpha，而非心理安慰。

### 9.1 处理流水线

```
新闻/研报原始文本（保留发布时间戳）
→ LLM 结构化抽取：
    {情绪极性: 利好/利空/中性, 强度: 1-5, 涉及个股: [...], 事件类型}
→ 过滤「中性」记录（中性新闻稀释信号，实测拉低 IC）
→ 时间戳对齐：t-1日15:00 ~ t日15:00 的新闻 → 归 t+1 交易日（防未来函数）
→ 时间衰减加权（越临近调仓日权重越高）
→ 按「股票×交易日」聚合 → 原始情绪得分
→ 与动量因子正交化（回归取残差）→ 纯情绪因子
→ AlphaLens：IC / 分层 / 换手检验 → 通过红线才纳入 LightGBM
```

### 9.2 IC 验收红线（防自欺的硬标准）

情绪因子必须满足以下四条全部通过才能进入实盘模型：

| 红线 | 标准 |
|------|------|
| IC 绝对值 | IC > 0.03 且符号稳定（全样本 IC 约 0.02，去中间组后约 0.04）|
| 分层单调 | Top 组显著优于 Bottom 组，且单调递进 |
| 正交化后仍显著 | 剥离动量后 IC 不能塌缩到 0 |
| 扣成本为正 | 情绪因子换手通常较高，需确认手续费后仍盈利 |

**第 2 月补充红线（6–8 因子定稿后）：** 与现有 6–8 因子组合的增量 IC 显著为正（与 v4.1 对齐）。

### 9.3 五个关键技术要点

1. **用便宜小模型批量打分。** 本地部署 Qwen 等开源模型或调用低价 API，控制成本。打分结果 Parquet 缓存，避免重复调用。
2. **必须正交化。** 情绪因子要剔除与动量因子重叠的部分（对动量回归取残差），否则只是「伪装成情绪的动量」，是伪 Alpha。
3. **严防未来函数。** 新闻时间戳必须用「可获得时间」，不能用事后修正版；节假日新闻加总到其后最近一个交易日。
4. **去中性 + 关注两端。** 情绪因子在得分两端分层效果最明显，去掉中间 50% 的股票后 IC 可成倍上升。
5. **相关度加权 + 时间衰减。** 情绪得分应按新闻与个股的相关度加权，并对时间做衰减——这是研究证明能显著提升因子表现的关键。

### 9.4 时序：第 1 月离线早止损

第 1 月（因子未定稿）先过前四条红线（IC > 0.03、分层单调、正交化后显著、扣成本为正）。若不通过 → **舆情路线整体后推至 M4+，不阻塞主策略上线**。第 2 月（6–8 因子定稿后）补第五条。

### 10.3 风控守卫唯一下单入口

```python
def place_order(ts_code, side, quantity, price, strategy_id):
    """唯一下单入口，必经风控守卫"""
    # 1. 风控前置校验（铁律一，不可绕过）
    risk_check_result = risk_guard(ts_code, side, quantity, price)
    if not risk_check_result.passed:
        log_risk_block(risk_check_result)
        return None
    # 2. 风控签名
    order = create_order(ts_code, side, quantity, price, strategy_id)
    order.risk_signature = risk_check_result.signature
    # 3. 幂等键去重（防重复下单）
    if is_duplicate(order.idempotency_key):
        return None
    # 4. 写入 execution_ledger
    execution_ledger.record(order)
    # 5. 调 xtquant 实际下单
    return xtquant.place_order(order)
```

| **B1 加仓判线（主裁决）** | 模拟盘 + 5 万阶段累计 ≥26 周：实测周度 rank-IC 均值 > 研究 IC − 1.0×SE；加仓前至少 13 周来自真实 5 万实盘 | 通过才允许 5 万→20 万加仓 |
| **B2 维持判线（终身滚动）** | 滚动 26 周实测 IC 均值 > 研究 IC − 1.645×SE 且 > 0 | 跌破→触发降仓复盘 |
| **B3 工程判线** | 成本偏差 ≤ +30%（实测/建模）、对账连续 4 周零差错、风控触发行为与状态机一致 | 与 B1 同为加仓必要条件（确定性验收）|

### 11.2 废除条款

以下旧闸门条款正式废除（QS-CAL-001 §一的 Monte Carlo 功效分析证明其数学上不可达）：
- v1.2 四子段递增 DSR（0.95/0.975/0.983/0.9875）；
- v1.3 合并段 DSR ≥ 0.95；
- 四子段离散度判据（max−min ≤ 0.8）。

validation 阶段查看预算 ≤5 轮（旧规保留）。

### 11.3 统计控制的设计哲学

在 test 段 104 周的周频数据上，**没有任何检验能可靠区分 SR=0 与 SR=0.8**（真实 SR=0.8 通过 DSR≥0.95 的概率仅 0.2%–3.5%）。真正的统计控制来自**资金分级本身**：5 万阶段在 25% 硬止损下最大损失被锁定在 1.25 万，相当于用有限可控的学费购买真实行为数据，让 B 类闸门在实盘分布上做最终裁决。

## §12 回测引擎

### 12.1 自研轻量事件驱动（选型决策）

**选型理由：** T+1 / 一手取整 / 涨跌停不可成交 / 动态滑点等 A 股特性，backtrader/vnpy 需大量魔改，魔改后黑箱比自研更难审计。

**双层互验架构：**
- **向量化回测**（快速因子研究）；
- **独立事件驱动撮合校验**（精确成本与约束）。

两层独立实现，结果不一致则停止，排查再上线。

### 12.2 成本参数

| 成本项 | 数值 |
|--------|------|
| 印花税（卖出）| 0.05% |
| 佣金 | 万 2.5（最低 5 元）|
| 过户费 | 万 0.1 |
| 回测滑点 | ≥ 0.2%（常态主池约 0.03%，1.6 万/5000 万下界）|
| 小盘股冲击成本 | 按真实成交额建模（不能假设瞬时全额成交）|

**进阶档（`cm_v3_advanced`，第 3 月起）：** 按分钟成交额/盘口深度/涨跌停距离建模（实盘下单集中在开盘/收盘/调仓窗口）。

### 12.3 A 股特性处理

- T+1 约束；
- 涨跌停不可成交；
- 100 股一手取整；
- 最小成交金额；
- 现金碎片处理；
- 部分成交处理；
- 开盘跳空；
- 高价股一手超 1.6 万剔除；
- 含退市/ST 防幸存偏差；
- 5 万试运行组合单独回测（5 万仅持约 5 只，与 Top10–15 不是同一策略，不能用 5 万阶段评估完整策略）。

### 12.4 验收指标监控

回测额外监控：最大回撤、回撤持续期、最大连续亏损次数；代码版本绑定回测结果（git commit hash 入 research_ledger）。

### 12.5 模型重训练协议

每季度重训练一次，数据窗口最近 5 年，重训练后用过去 12 个月冻结测试集验证；上线条件 = 验证集夏普 > 0.6 且 > 当前线上模型 90%；不通过则保留线上模型，人工审查再决策。

## §14 部署章

### 14.1 家用 Windows 11 PC 拓扑

**主机配置（推荐）：**

| 组件 | 要求 | 说明 |
|------|------|------|
| CPU | 6 核 + | 研究回测峰值 CPU 密集 |
| 内存 | 32 GB | 研究回测峰值内存 > 16 GB |
| 存储 | 1 TB NVMe | DuckDB + Parquet + SQLite |

| 网络备链路 | **手机热点** | 主链路故障自动/手动切换 |

### 14.2 进程拓扑（五个常驻/定时进程）

| 进程编号 | 名称 | 运行时段 | 说明 |
|---------|------|---------|------|
| ① | QMT/迅投终端 | 交易日常驻 | xtquant 依赖此终端在线 |
| ② | 执行守护进程（xtquant）| 开盘时段（9:15–15:00）| 风控守卫 + 下单 + 状态机 |
| ③ | 盘后数据管道 | 每日 17:00（任务计划）| AKShare/Tushare/BaoStock 采集 → 入库 → visible_at 打戳 |
| ④ | 研究环境（Jupyter/脚本）| **非交易时段** | 因子研究/回测，禁止在交易时段运行（资源竞争） |
| ⑤ | 监控告警进程 | 常驻（独立）| **watchdog 互查**：监控自身 + 进程②存活 |

**watchdog 设计原则：** 监控进程⑤必须与执行进程②互相监控（不是单向），确保监控进程自身故障也能被发现。

### 14.3 告警设计

**推送渠道：** Server 酱 / 钉钉机器人 → 推送到手机。

**关键告警事件（必须触发推送）：**

| 告警类型 | 严重级别 | 处置 |
|---------|---------|------|
| 风控触发（任意级别）| 🔴 高 | 立即检查，20%以上触发 = 人工介入 |
| 下单失败 | 🔴 高 | 检查 xtquant 连接 + 资金状态 |
| 数据管道失败 | 🟡 中 | 次日补采；重要日期（财报/除权）升级为高 |
| 进程失联（②或⑤）| 🔴 高 | 立即检查，可能需 break-glass |
| UPS 切换至电池 | 🟡 中 | 准备手动清仓；电池约 10 分钟 |
| 对账差错 | 🟡 中 | 当日排查，连续 4 周 → B3 判线失败 |

### 14.4 备份策略

| 备份对象 | 频率 | 目标 | 工具 |
|---------|------|------|------|
| SQLite（execution_ledger + 交易审计）| 每日盘后增量 | 外置硬盘 + 云盘 | rclone |
| DuckDB（因子/行情面板）| 每日盘后增量 | 外置硬盘 + 云盘 | rclone |
| Parquet 文件 | 每日盘后增量 | 外置硬盘 | rsync |
| 代码 | 每次提交 | Git + 远程仓库（GitHub/Gitee）| git push |
| 周度恢复演练 | 每周 | — | 从备份恢复最新 SQLite，验证可读 |

### 14.5 灾备 SOP

**PC 故障场景（中断自动交易）：**

1. **发现：** 监控告警推送进程失联 → 手机收到告警；
2. **立即：** 打开手机券商 APP，检查当前持仓状态；
3. **评估：** 若市场波动正常 → 保持持仓，等待 PC 恢复；若市场异常（如大盘急跌） → 执行 break-glass 手动清仓 SOP；
4. **恢复：** 修复/重启 PC，从 rclone 备份恢复最新 SQLite + DuckDB，重启进程拓扑；
5. **验收：** 持仓对账 + 数据完整性检查 + 进程健康检查全绿后恢复自动交易；
6. **目标：** T+1 交易日内恢复自动化（不超过一个交易日停摆）。

### 14.6 运维预算

| 运维项 | 时间 | 频率 |
|--------|------|------|
| 每日盘后巡检 | 5–10 分钟 | 每个交易日 |
| 周度对账复盘 | 2 小时 | 每周末 |
| 周度备份恢复演练 | 15 分钟 | 每周 |
| **合计** | **≤4h/周** | — |

> **超出 4h/周 即触发简化评审**：首选砍范围（先砍 LLM 舆情至 M4+，再砍 LightGBM 留线性），不要靠增加运维时间维持过多功能。

## §17 验收指标

### 17.1 回测双档验收

| 指标 | 现实档（上实盘硬线）| 理想档（追求目标）|
|------|-------------------|-----------------|
| 年化收益 | 10–15%（样本外、扣成本）| 15–22% |
| 最大回撤 | < 25%（硬线，与 25% 止损一致）| < 20% |
| 夏普比率 | > 0.8 | > 1.0 |
| Calmar 比率 | > 0.5 | > 0.8 |
| 样本外检验 | 必做（walk-forward，最近 1–2 年）| 必做 |

**线性基线口径：** 夏普 ≥ 0.6 进入研究下一步；夏普在 0.6–0.8 之间须靠 LightGBM 增量补足到 0.8 才上实盘；夏普 ≥ 0.8 才可上实盘。

### 17.2 防过拟合铁律

1. 严格区分样本内/外，参数只在样本内优化；用 walk-forward（滚动训练-测试），而非单次切分；
2. 滑点至少按 0.2% 估计，手续费按真实成本（印花税 + 佣金 + 过户费），小盘股建模冲击成本；
3. 因子数量克制，宁少勿滥；LightGBM 用中等深度 + L1/L2 正则 + 限制最小叶子样本数；
4. 回测样本必须包含已退市/已 ST 股票（防幸存者偏差）；
5. 等价独立样本约 20–50 万（决定模型复杂度上限），别被「千万行面板」迷惑。

### 17.3 数据质量闸门

| 检查项 | 通过标准 |
|--------|---------|
| 行情覆盖率 | 每交易日 ≥ 99% |
| 退市/ST 覆盖率 | 历史退市股完整可查 |
| 复权因子完整性 | 缺失率 = 0 |
| 数据新鲜度 | 盘后 2h 内入库 |

## 附录B：三条最重要原则

**一、先活下来，再谈盈利。**  
25% 硬止损是生命线，必须工程化、自动化、不可手动绕过。把风控做成下单接口的唯一守卫，加物理一键熔断。降风险 + 恢复规则 + 子策略预算 + 停盘规则 + 极端场景兜底 + 熔断失败模式分析，每一项都是「先活下来」的组成部分。

**二、统计正确优先于一切。**  
多重检验对全因子族做、试验次数记账、测试集物理隔离、Deflated Sharpe——避免完美运行一个数据窥探出来的无效策略。在 2 年周频数据上，任何检验都无法可靠区分 SR=0 与 SR=0.8；统计控制的真正手段是资金分级本身。

**三、可开工优先于再设计。**  
设计已达单人量化准专业天花板，继续加设计边际递减。重心转向四份可开工宪法文档（QS-C02 至 QS-C05）+ 三个月内模拟盘全链路上线。模型上不贪心：LightGBM + LLM 舆情足矣，不上 RL/DL。
