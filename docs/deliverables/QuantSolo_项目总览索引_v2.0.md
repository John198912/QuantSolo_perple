## 四、宪法修订流程（v2.0 起生效）

1. **触发**：审查发现矛盾 / 实盘事实推翻假设 / 监管变更 / 闸门重定标需求。
2. **提案**：写明「现条文 → 问题 → 新条文 → 影响面（受影响文档清单）」。
3. **证据**：涉及闸门数字的修订**必须**附重跑后的 `gate_calibration.py` 功效表（QS-CAL-001 元规则）。
4. **冷静期**：非紧急修订设 48 小时冷静期（一人公司防冲动改宪法）；紧急合规修订除外。
5. **落地**：受影响文档全部出新全文版（禁止增量补丁），更新本索引 §二 版本锁定矩阵，Git 打 tag。
6. **频率约束**：宪法修订 ≤1 次/月（紧急合规除外）；高于此频率说明设计不稳，先停手复盘。
