## 〇、给 Agent 的一句话定位

你在为一家**一人公司**实现一套 A 股周度中低频量化交易系统（20 万本金、家用 Windows 11 PC、QMT/xtquant 实盘通道）。这个系统的第一性约束不是性能、不是优雅，而是：**风控不可绕过、数据点时正确、行为可审计、单人可运维**。你写的每一行代码都可能在无人盯盘时真实下单——按这个标准要求自己。

## 二、不可违背约束（Agent 红线）

以下任何一条被违反，该次提交视为失败，无论功能是否正常。

### R1 风控守卫是唯一下单入口
- 所有真实/模拟下单**必须**经过 `risk_guard` 守卫层（E03 §6 的装饰器/守卫模式）。
- 禁止在任何代码路径（包括测试工具、调试脚本、回填脚本）中直接调用 xtquant 下单 API。唯一例外：`break_glass.py` 物理熔断脚本（其本身就是越过策略层的设计，见 C04 §五）。
- 必须保留并通过「守卫旁路不可达」测试（E04 §3）：静态扫描 `import xtquant` 只允许出现在 `src/execution/adapters/xtquant_adapter.py` 与 `tools/break_glass.py`。

### R2 点时正确性
- 一切研究/回测数据读取必须走 canonical 查询（四键排序：`visible_at DESC, revision_seq DESC, snapshot_rank DESC, record_id DESC`），禁止绕过点时引擎直接 SELECT 原始表。
- 修改=追加 ACTIVE，撤销=追加 VOIDED，**任何表禁止 UPDATE/DELETE 历史行**。
- 新增任何数据管道必须同时实现 visible_at 赋值规则（C03 §10）并补充对应回归场景。
- pytest 13 场景（C03 §12）是底座：任何涉及数据层的 PR，13 场景必须全绿才能合并。

### R3 冻结参数不可改
以下数字属于宪法，Agent 在任何情况下不得修改、不得"顺手优化"（修改须走 C00 §四宪法修订流程，那是人类的事）：
- 闸门数字：A1（合并段夏普≤0 硬否决）、A2（DSR<0.5 降级，N_eff≤6）、B1（26 周窗，IC>研究IC−1.0×SE）、B2（−1.645×SE 且>0）、B3（成本偏差≤+30%、4 周零差错）
- 风控数字：三级回撤 20%/25%、单票≤8%、单行业≤30%、日均成交额≥5000 万、内部限额 1 笔/秒、200 笔/日
- 数据切分：train 2016–2021 / validation 2022–2023 / test 2024–2025
- 成本参数：印花税卖出 0.05%、佣金万 2.5（最低 5 元）、过户费万 0.1、滑点≥0.2%
- 这些参数在代码中必须集中在**冻结配置文件**（E02 §5：`config/frozen_params.yaml`），加载时校验 SHA256 与宪法登记值一致，篡改即启动失败。

### R4 研究纪律由代码强制
- test 段（2024–2025）数据在 M2 的研究代码中默认**物理不可达**（独立目录+权限/路径隔离，E02 §3），只有 `run_test_evaluation.py` 一个入口可以触达，且每次运行强制写入试验登记表（C02 §五）并打印剩余 N_eff 预算。
- 禁止在任何探索性脚本、notebook、单测 fixture 中引用 test 段日期范围的真实数据。

### R5 状态机与对账
- 订单/持仓状态只能取 C04 §一 的 15 个运行态枚举；新增状态=宪法修订，禁止。
- 每笔委托必须携带 `order_remark = client_order_id`（幂等键，C04 §八）；对账三方口径（本地账本/券商回报/券商持仓）的实现不得合并简化。

### R6 诚实工程
- 禁止为让测试变绿而放宽断言、mock 掉风控、跳过场景（`skip`/`xfail` 须附 issue 编号与人类批准记录）。
- 禁止引入 PRD §10 反范围内容：RL/DL 模型、PostgreSQL/ClickHouse、高频/日内策略、未报备的下单通道。
- 所有金额/费率计算用 `Decimal` 或整数分，禁止 float 直接算钱（E03 §8）。

## 三、实施顺序与任务协议

### 3.1 总顺序（来自 QS-E06，不得跳序）

```
M1 数据基建（~60h）：仓库骨架 → 三源适配器 → 点时引擎+9表 → 13场景全绿
                     → 盘后管道任务计划 → visible_at 重放验证
M2 研究+执行（~90h）：因子库 → 线性基线(validation 夏普≥0.6) → walk-forward
                     → LightGBM 增强 → [人类执行 test 评估] 
                     ∥ 并行：状态机15态 → 风控守卫 → xtquant适配器 → 拆单/重试
M3 模拟盘上线（~60h）：对账器 → 监控告警 → 物理熔断 → 模拟盘空跑
                     → T1–T13 演练 → 4周零差错追踪开始
```

依赖铁则：**点时引擎未过 13 场景，禁止开始因子开发；风控守卫未过旁路不可达测试，禁止接通 xtquant 适配器；对账器未上线，禁止模拟盘自动循环。**

### 3.2 单个任务的标准工作流（每次会话遵循）

1. **取任务**：从 QS-E06 当周任务表取一项（或人类指定）；在 issue 中登记。
2. **加载上下文**：按 §1.2 表加载最小文档集；读 E03 中该模块的签名与伪代码。
3. **写测试先行**：先把 E04 对应验收用例落成失败的测试，再实现。
4. **实现**：遵守 E02 仓库结构与接口 ABC，不发明新架构；偏离 E03 签名需在 PR 说明中标注「设计偏离」及理由。
5. **自检**（§3.3 命令）全绿。
6. **提交**：Conventional Commits（`feat(data): ...` / `fix(risk): ...`）；PR 描述含：对应 E06 任务编号、触碰的宪法条款（只读引用）、测试证据、设计偏离清单。
7. **汇报**：向人类输出 ≤10 行摘要：做了什么/验证了什么/遗留什么/下一步建议。

### 3.3 标准自检命令（每次提交前必跑）

```bash
pytest tests/pit -x          # 13 场景（数据层改动必跑，全量回归每周必跑）
pytest tests/unit            # 单元层（含 15 态覆盖、守卫旁路不可达）
python tools/static_guard_scan.py   # 静态扫描：xtquant import 白名单 / UPDATE/DELETE 禁令 / float 算钱
python tools/frozen_params_check.py # 冻结参数 SHA256 校验
pytest tests/integration -m "not live"  # 集成层（不触真实通道）
```

任何一项失败禁止提交。CI 等价物为 Windows 任务计划每日全量（E04 §10）。

## 四、人机权限边界

### 4.1 Agent 禁止执行（必须停下请求人类）

1. 触发 test 段评估（消耗 N_eff 预算，全生命周期仅 6 次）
2. 修改宪法文档或冻结参数（包括"显然是笔误"的情况——报告，不要修）
3. 真实资金操作、券商开户/报备材料提交、任何对券商系统的写操作
4. 删除/重写历史数据表、清空试验登记表、修改审计日志
5. 安装未列入 E02 附录 A 依赖清单的新第三方库（先提案）
6. 砍范围决策（E06 §10 的三刀顺序由人类下令执行）

### 4.2 Agent 应主动做（无需请示）

1. 发现文档矛盾/缺陷 → 写入 `docs/issues_found.md` 并在汇报中置顶
2. 发现测试覆盖缺口 → 补测试
3. 发现潜在未来函数/数据泄漏 → 立即标红上报（这是本项目最高优先级缺陷类别）
4. 重构以符合 E02/E03 既有设计（行为不变 + 测试全绿前提下）
5. 维护 `CHANGELOG.md` 与模块级 README

### 4.3 升级路径

阻塞 >30 分钟（依赖缺失、文档矛盾、外部服务不可用）→ 停止猜测，输出「阻塞报告」：现象 / 已尝试 / 候选方案 A/B / 建议。**宁可停，不可猜**——尤其在风控、点时、资金计算三个领域。

## 五、代码规范速查（细则见 QS-E02 §5–7）

- Python 3.11+，全量 type hints，`ruff + mypy` 零告警合并
- 策略逻辑层=纯函数（无 IO、无全局态、可独立单测）；IO 只存在于适配器层
- 日志：结构化 JSON（E02 §6），交易路径每个状态转换必有一条含 `client_order_id` 的日志；禁止 `print`
- 异常：交易路径禁止裸 `except`；按 E02 §7 分类（可重试/需人工/致命→HALT）
- 时间：内部一律 `Asia/Shanghai` aware datetime；与交易所/数据源边界处显式转换
- 随机性：一切研究代码显式传种子；回测结果必须可逐字节复现（C02 §九）
- 配置：冻结参数（frozen_params.yaml，只读校验）与可调参数（settings.yaml）物理分离
